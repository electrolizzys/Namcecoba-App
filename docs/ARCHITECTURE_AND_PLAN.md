# Namtsetsoba — გეგმა და არქიტექტურა (ჩვენთვის)

ეს ფაილი არის სახელმძღვანელო ორი ადამიანისთვის, არა ლექტორისთვის. როცა დაიბნევით, აქ უნდა დაბრუნდეთ.

რა ვაკეთებთ მოკლედ: აპი სადაც ობიექტი ყიდის დარჩენილ/ზედმეტ საკვებს კალათებად, მომხმარებელი ხედავს შეთავაზებას, ყიდულობს აპში, მერე მიდის ასაღებად და აჩვენებს კოდს.

ვინ არის სისტემაში

- ჩვეულებრივი მომხმარებელი — რეგისტრაცია აპში, ყიდულობს, ხედავს შეკვეთებს.
- ობიექტი (ბიზნესი) — თქვენი გადაწყვეტით არ ირეგისტრირება პირდაპირ; ჯერ განხილვა/დადასტურება, მერე ანგარიში.
- ადმინი პირველ ეტაპში შეიძლება იყოთ თქვენ ორივე + Supabase-ის ვებ პანელი, ცალკე დიდი ადმინ პანელი არ არის საჭირო დასაწყისისთვის.

რა გვინდა პირველ რიგში (MVP)

- შესვლა / რეგისტრაცია მომხმარებლისთვის.
- სია ან ფიდი ცოცხალი კალათების (გვერდებად ჩატვირთვა, არა ყველაფერი ერთად).
- დალაგება: ფასით და სურვილის შემთხვევაში მანძილით (ლოკაციის ნებართვა საჭიროა მანძილზე რომ იმუშაოს).
- კალათის დეტალური გვერდი: მაღაზია, დრო, ფასი, რა არის შიგნით.
- შეკვეთის ნაკადი: გადახდის ეკრანი (იხილე ქვემოთ დემო vs რეალური) და მერე შეკვეთა პროფილზე.
- პროფილი: რა გაქვს აქტიური, ისტორია, უნიკალური კოდი ასაღებად.
- ძებნა მაღაზიის სახელით და ფილტრი კატეგორიით.

რა შეგვიძლია მერე, როცა ზემოთ მუშაობს

- რუკა MapKit-ით (იგივე მონაცემები რაც სიაში, უბრალოდ პინები).
- ფავორიტები და პრიორიტეტი ხშირად შეკვეთილ ობიექტებზე.
- პუშ ნოთიფიკაციები.
- ჩატი ობიექტთან — ეს ყველაზე დიდი სამუშაოა, ბოლომდე გადაიტანეთ თუ დრო არ გაქვთ.

გადახდა — რა გინდათ პროდუქტით და რა არის რეალობა

იდეა: მომხმარებელი იხდის კალათისთვის აპში, თქვენ იღებთ პროცენტს. ეს ნორმალური მოდელია.

რეალური ფული აპში ნიშნავს: ბანკი/გადამხდელი სისტემა (ხშირად Stripe ან სხვა), ხელშეკრულებები, გადასახადები, თაღლითობის თავიდან აცილება, და Apple-ის წესები რას ყიდით. ეს სტუდენტური პროექტისთვის ხშირად ძალიან დიდი ტვირთია.

დემო რეჟიმი (რასაც თქვენ გინდათ ახლა)

- აპში არის ღილაკი მაგალითად გადახდა დასრულებულია ან იმიტაცია.
- სერვერზე შეკვეთა მაინც იქმნება სტატუსით paid ან demo_paid რომ მთელი ნაკადი იყოს სწორი.
- პროცენტი მათემატიკურად შეგიძლიათ შეინახოთ ცხრილში (რამდენი იყო კალათა, რამდენი თქვენი საკომისიო) მაგრამ ფული რეალურად არ გადაირიცხება.

როცა მოგინდებათ რეალური გაყიდვა

- ერთი სერვისი Checkout ან PaymentService კოდში, არა გაფანტული ლოგიკა ყველა ეკრანზე.
- გადახდის დადასტურება უნდა მოდიოდეს სერვერიდან (webhook), არა მხოლოდ იმისგან რომ აპმა თქვა გავიდა.

რა არის Supabase მარტივად

Supabase არის მზა ბექენდი ინტერნეტში: ბაზა (PostgreSQL), შესვლა/რეგისტრაცია, ფაილების საცავი სურათებისთვის, და შესაძლებელია realtime განახლებებიც. თქვენ წერთ iOS აპს და წესებს ბაზაში ვინ რას ხედავს. ასე პატარა აპებიც კი აკეთებენ, არა მხოლოდ გიგანტები.

მონაცემები სად ზის და რატომ არ ტყდება ათასობით იუზერზე

- ჭეშმარიტება ყოველთვის ბაზაშია, არა მხოლოდ ტელეფონში.
- სიები იტვირთება ნაწილ-ნაწილ (მაგალითად 20 ჩანაწერი), არა მთელი კატალოგი ერთად.
- ბაზაში ინდექსები იმ სვეტებზე სადაც ფილტრავთ და ალაგებთ.
- სურათები Storage-ში, ბაზაში მხოლოდ გზა ან იდენტიფიკატორი.

რუკა მოკლედ

- ახლოს პირველი შეიძლება იყოს ჩვეულებრივი სია დალაგებული მანძილით, რუკა არ არის აუცილებელი დასაწყისში.
- როცა დაამატებთ, iOS-ზე ჩვეულებრივია Apple MapKit. მაღაზიას უნდა ქონდეს გრძედი და განედი ბაზაში.

MVVM რას ნიშნავს პრაქტიკაში

- ეკრანი SwiftUI-ში მარტივია.
- ViewModel ინახავს მდგომარეობას და იძახებს სერვისებს.
- სერვისები საუბრობენ Supabase-თან. ასე ლოგიკა არ იშლება ყველა ფაილში.

რა ტექნიკა სად გამოვიყენოთ

- SwiftUI — ყველა ეკრანი
- MVVM — სტრუქტურა
- async await — ქსელის ოპერაციები
- Supabase Swift SDK — Swift Package Manager-ით
- Supabase Auth — შესვლა
- PostgreSQL Supabase-ში — მონაცემები
- Row Level Security Supabase-ში — ვინ რომელ ჩანაწერს ხედავს, ეს მნიშვნელოვანია როცა აპი საჯაროა
- Supabase Storage — ფოტოები
- MapKit + CoreLocation — რუკა და ლოკაცია მერე
- ქრაშებისთვის მოგვიანებით Crashlytics ან Sentry ერთი აირჩიეთ, არა ორივე ჯერ ერთად

ბაზაში რoughly რა ცხრილები (სახელები შეგიძლიათ შეცვალოთ)

- profiles — ვინ არის, როლი customer თუ business
- stores — ობიექტი, მისამართი, კოორდინატები, დადასტურებული თუ არა
- baskets — კალათა, ფასი, რაოდენობა, pickup დრო, სტატუსი
- orders — ვისია, რომელი კალათა, სტატუსი, pickup კოდი, თანხა და სურვილის შემთხვევაში საკომისიო ველები დემოსთვისაც
